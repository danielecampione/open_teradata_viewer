How to create a volatile table without generating errors

To create a volatile table without generating the error "Only an ET or null statement is legal after a DDL Statement", do a COMMIT after the creation of the volatile table and before submitting any SQL instruction that acts on the volatile table. Do a commit after populating the same table to be able to use it.

How to install a plugin

for Linux users:
To install a plugin download the compressed archive from the official repository and extract the content to the user home directory. Run Open Teradata Viewer and select the Files comparator.. menu voice.

for Windows users:
To install a plugin download the compressed archive from the official repository and extract the content to the same directory where Open_Teradata_Viewer_x.xx.jar resides. Run Open Teradata Viewer and select the Files comparator.. menu voice.
